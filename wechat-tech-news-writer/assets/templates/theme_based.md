## 内容组织指南

### 主题分类原则

#### 何时使用主题分类?

✅ **推荐场景**:
- 多公司参与同一项目(如政府合作计划)
- 行业性趋势(如AI投资热潮)
- 技术领域进展(如量子计算突破)
- 政策影响多个企业(如出口管制)

❌ **不推荐场景**:
- 需要强调地域差异时
- 政治敏感度较高的内容
- 读者更关心国内外对比

### 跨公司新闻整合

**合并示例**:

原始新闻(公司分类):
```
OpenAI: 获得政府合同1.5亿美元
Anthropic: 获得政府合同1.8亿美元
Google: 获得政府合同2亿美元
xAI: 获得政府合同0.7亿美元
```

主题整合后:
```
### 政府与企业合作

**美国政府6亿美元AI研发合作计划**

美国政府宣布向四家AI公司分配总计6亿美元研发项目:
- OpenAI: $1.5亿,聚焦对话AI系统
- Anthropic: $1.8亿,专注AI安全研究
- Google: $2亿,提供云计算基础设施
- xAI: $0.7亿,探索多模态应用

该计划标志着政府AI应用进入加速期...
```

**整合优势**:
- 避免重复描述背景
- 突出整体规模和影响
- 便于横向对比
- 减少总字数

### 主题分类技巧

#### 1. 识别共同主题

寻找新闻之间的共同点:
- **投资主题**: 融资、IPO、研发支出、并购
- **技术主题**: 产品发布、技术突破、专利申请
- **市场主题**: 财报、股价、市场份额、竞争格局
- **政策主题**: 监管法规、合规要求、政府支持
- **人才主题**: 招聘、高管变动、人才竞争

#### 2. 创建子分类

每个主题下设置合理的子分类:

```
🤖 AI公司动态
├── 政府合作 (跨公司合作项目)
├── 财报业绩 (季度/年度财报)
├── 产品发布 (新产品、新功能)
├── AI应用 (行业应用案例)
├── 融资投资 (融资轮次、投资交易)
└── 监管争议 (合规、争议事件)
```

#### 3. 避免过度细分

❌ **过度细分**:
```
### OpenAI动态
### Anthropic动态
### Google AI动态
### Microsoft AI动态
```

✅ **合理聚合**:
```
### 政府与企业合作
(包含所有企业与政府的合作项目)

### 企业营收与市场表现
(包含多家企业的财报和市值数据)
```

### 合规优化要点

#### 政府合作项目表述

**高风险内容**:
- 军事合同
- 国防项目
- 安全应用

**优化方法**:
- ❌ "五角大楼AI军事合同"
- ✅ "美国政府AI研发合作计划"
- ❌ "国防部采购AI武器系统"
- ✅ "公共部门AI技术应用项目"

#### 中国企业报道

**避免对抗性表述**:
- ❌ "中国企业遭受技术封锁"
- ✅ "中国企业加速自主研发进程"
- ❌ "美国打压中国科技公司"
- ✅ "国际贸易政策调整,产业链重构"

#### 平衡报道

在每个主题下保持平衡:
- 正面进展 + 挑战困难
- 国内企业 + 国际企业
- 政策支持 + 监管要求
